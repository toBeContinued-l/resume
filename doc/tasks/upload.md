# 文件上传模块任务

## 最小可执行任务

- [x] 定义上传请求处理边界：单次只允许上传 1 个文件。
- [x] 实现上传 API `/api/resumes/upload` 的登录态校验。
- [x] 实现文件扩展名校验，仅允许 `.doc`、`.docx`、`.pdf`。
- [x] 实现 MIME 类型校验，并与扩展名做一致性检查。
- [x] 实现文件大小校验，单个文件不超过 15MB。
- [x] 实现当前用户未删除简历数量检查，达到 3 份时返回 `RESUME_LIMIT_REACHED`。
- [x] 生成服务端安全文件名，原始文件名只作为元数据保存。
- [x] 调用临时文件存储服务创建任务目录并保存原始文件。
- [x] 创建 `resumes` 记录，初始状态为 `generating`。
- [x] 创建 `generation_tasks` 记录，初始状态为 `pending`。
- [x] 投递生成任务消息到队列。
- [x] 返回 `resumeId`、`taskId` 和初始任务状态。
- [x] 实现上传失败回滚：删除已写入临时文件，不保留失败简历记录和任务记录。
- [x] 实现上传页面的文件选择、格式提示、大小提示和 3 份限制提示。
- [x] 实现上传成功后的任务进度跳转或轮询入口。
- [x] 编写上传单元测试：未登录、超限大小、不支持格式、3 份限制、成功创建任务。
- [x] 编写失败回滚测试：临时文件写入失败或队列投递失败时不保留用户可见记录。

## 完成标准

- [x] 合法文件上传后能创建简历记录、任务记录和队列消息。
- [x] 非法上传不会留下失败记录。
- [x] 上传数量限制只统计未删除简历。
