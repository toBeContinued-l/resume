# 临时文件存储模块任务

## 最小可执行任务

- [x] 定义 `TEMP_UPLOAD_ROOT` 环境变量和默认本地临时目录策略。
- [x] 实现 `createTaskDir`，按 `{userId}/{taskId}` 创建任务目录。
- [x] 实现 `saveOriginal`，保存原始上传文件并返回 `TempFileRef`。
- [x] 实现 `getTaskDir`，根据用户和任务 ID 返回受控任务目录。
- [x] 实现 `removeTaskDir`，只允许删除任务目录。
- [x] 实现解析资源目录 `assets/` 的创建和写入能力。
- [x] 实现 `.doc` 转换结果 `converted.docx` 的保存引用。
- [x] 实现路径归一化和根目录校验，防止路径穿越。
- [x] 确保原始上传文件不生成公网 URL。
- [x] 确保前端响应和持久化简历内容不包含原始临时文件路径。
- [x] 实现删除不存在任务目录时的幂等处理。
- [x] 编写路径安全测试：`../`、绝对路径、伪造 userId 或 taskId 都不能逃逸根目录。
- [x] 编写目录生命周期测试：创建、写入、读取、删除。
- [x] 编写删除幂等测试：重复删除不影响其他任务目录。

## 完成标准

- [x] 所有临时文件读写都被限制在 `TEMP_UPLOAD_ROOT` 下。
- [x] 上传、解析、转换和清理流程可复用同一服务接口。
- [x] 临时文件不会被公开访问或长期依赖。
